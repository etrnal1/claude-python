# CLAUDE.md — AI Assistant Guide for claude-python

## Repository Overview

This repository is a **production build** of a Progressive Web Application (PWA) for offline knowledge base management. It is a mobile-first app primarily targeting iPhone, built with **Vue.js 3**, **Vite**, and standard browser PWA APIs.

> **Important**: The repo contains only the compiled/bundled output — no source `.vue`, `.ts`, or `.js` source files are present. All editable logic lives inside the built bundles under `assets/`.

---

## Tech Stack

| Layer | Technology |
|---|---|
| Framework | Vue.js 3.5.30 (Composition API) |
| Build tool | Vite (hash-based asset versioning) |
| Language | TypeScript (compiled away; not present in repo) |
| Persistence | IndexedDB via Dioxie wrapper |
| PDF rendering | PDF.js (pdfjs-dist) |
| Offline support | Service Worker (Cache API) |
| Platform target | iOS / modern browsers |

## Application Features

The app exposes a tab-based interface with the following views:

- **Personal Notes** (`PersonalNotes` component): Full CRUD for notes with categories, tags, starring, expiry, search, and word-count stats.
- **Surge/Proxy Config** (`SurgePage` component): Generator for Surge and Clash network proxy configuration files.
- **Backup/Restore**: Export and import IndexedDB data.
- **Settings**: App preferences stored in LocalStorage.
- **Diagnostics**: In-app troubleshooting tools.
- **Trash**: Soft-deleted notes recovery.

## Service Worker Caching

**File:** `sw.js` — Cache version: `kb-pwa-v11`

- **Static assets** (JS/CSS/icons): Cache-first strategy.
- **Navigation requests**: Network-first with offline HTML fallback.
- **Updates**: Background update without forced reload.
- **Max file size**: 50 MB (`50 * 1024 * 1024` bytes).

When bumping the SW cache, increment `CACHE_VERSION` in `sw.js` and update the precache URL list to match new asset hashes.

## Development Notes

Since this repo contains only the build output:

1. **No source files to edit directly.** Any real changes require the original source code.
2. **No package.json or node_modules** — dependencies are compiled into the bundle.
3. **No tests** — no test infrastructure is included in this distribution.
4. **No CI/CD pipelines** are configured.

To rebuild:
- Requires the original Vite project (Vue SFCs + TypeScript sources)
- Run `vite build` to produce new hashed asset files
- Update `sw.js` precache list with new filenames and increment `CACHE_VERSION`

## Branch Workflow

- **`main`**: Stable production build.
- **`claude/...`**: AI-assisted feature branches — merge to `main` when complete.

Commit messages should be concise and describe *what changed in the build* (e.g., `Update SW cache version to kb-pwa-v12`, `Add icon-1024 to PWA manifest`).
